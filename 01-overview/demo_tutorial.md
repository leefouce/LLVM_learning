# `sum_to` 的 LLVM IR 逐行讲解

这份文档只讲 `01-overview/hello.c` 里的 `sum_to` 函数，并且按下面这种格式解释：

```text
IR 代码  ->  解释
```

原始 C 代码是：

```c
int sum_to(int n) {
    int s = 0;
    for (int i = 1; i <= n; i++) {
        s += i;
    }
    return s;
}
```

直观含义：

```text
如果 n >= 1，计算 1 + 2 + ... + n
如果 n < 1，循环一次都不执行，返回 0
```

## 1. 未优化 IR 逐行讲解

未优化 IR 最大特点是：它很像把 C 变量都放进内存里反复 `load` / `store`。

先记住这几个对应关系：

```text
alloca = 开一个变量槽
store  = 写变量
load   = 读变量
br     = 跳转
ret    = 返回
```

在这个函数里：

```text
%0  = 参数 n 的值

%2  = n 在栈上的地址
%3  = s 在栈上的地址
%4  = i 在栈上的地址

%5  = for 循环的条件判断块
%9  = for 循环的身体：s += i
%13 = for 循环的更新：i++
%16 = 循环结束后的 return
```

```llvm
define dso_local i32 @sum_to(i32 noundef %0) #0 {
```

-> 定义一个函数 `sum_to`。  
-> 返回值类型是 `i32`，也就是 32 位整数，类似 C 里的 `int`。  
-> 参数是 `i32 %0`，这里 `%0` 就是 C 里的 `n`。  
-> 对应：

```c
int sum_to(int n) {
```

---

```llvm
  %2 = alloca i32, align 4
```

-> 在栈上分配一个 `i32` 空间。  
-> `%2` 是这个空间的地址。  
-> 这里用来保存参数 `n`。  
-> 可以理解成：

```c
int n_addr;
```

---

```llvm
  %3 = alloca i32, align 4
```

-> 在栈上分配一个 `i32` 空间。  
-> `%3` 是这个空间的地址。  
-> 这里用来保存变量 `s`。  
-> 对应：

```c
int s;
```

---

```llvm
  %4 = alloca i32, align 4
```

-> 在栈上分配一个 `i32` 空间。  
-> `%4` 是这个空间的地址。  
-> 这里用来保存变量 `i`。  
-> 对应：

```c
int i;
```

---

```llvm
  store i32 %0, ptr %2, align 4
```

-> 把 `%0` 的值存到 `%2` 指向的内存里。  
-> `%0` 是参数 `n`。  
-> `%2` 是保存 `n` 的地址。  
-> 对应：

```c
n_addr = n;
```

---

```llvm
  store i32 0, ptr %3, align 4
```

-> 把整数 `0` 存到 `%3` 指向的内存里。  
-> `%3` 是变量 `s` 的地址。  
-> 对应：

```c
s = 0;
```

---

```llvm
  store i32 1, ptr %4, align 4
```

-> 把整数 `1` 存到 `%4` 指向的内存里。  
-> `%4` 是变量 `i` 的地址。  
-> 对应：

```c
i = 1;
```

---

```llvm
  br label %5
```

-> 无条件跳转到基本块 `%5`。  
-> `%5` 是循环条件判断的位置。  
-> 对应：

```c
goto loop_condition;
```

---

```llvm
5:                                                ; preds = %13, %1
```

-> 这是一个基本块，名字是 `%5`。  
-> 它是循环条件判断块。  
-> `preds = %13, %1` 是注释，表示这个块可以从两个地方跳过来：

```text
%1  = 函数入口初始化之后
%13 = i++ 之后跳回来
```

---

```llvm
  %6 = load i32, ptr %4, align 4
```

-> 从 `%4` 指向的内存里读取一个 `i32`。  
-> `%4` 是变量 `i` 的地址。  
-> 所以：

```text
%6 = i
```

---

```llvm
  %7 = load i32, ptr %2, align 4
```

-> 从 `%2` 指向的内存里读取一个 `i32`。  
-> `%2` 是参数 `n` 的地址。  
-> 所以：

```text
%7 = n
```

---

```llvm
  %8 = icmp sle i32 %6, %7
```

-> 比较 `%6 <= %7`。  
-> `%6` 是 `i`。  
-> `%7` 是 `n`。  
-> `icmp` 表示整数比较。  
-> `sle` 表示 signed less or equal，也就是“有符号小于等于”。  
-> 对应：

```c
i <= n
```

-> 比较结果存在 `%8` 里，类型是 `i1`，也就是 true / false。

---

```llvm
  br i1 %8, label %9, label %16
```

-> 根据 `%8` 的真假跳转。  
-> 如果 `%8` 为 true，跳到 `%9`。  
-> 如果 `%8` 为 false，跳到 `%16`。  
-> 对应：

```c
if (i <= n)
    goto loop_body;
else
    goto loop_exit;
```

---

```llvm
9:                                                ; preds = %5
```

-> 这是基本块 `%9`。  
-> 它是循环体。  
-> 对应 C 里的：

```c
{
    s += i;
}
```

---

```llvm
  %10 = load i32, ptr %4, align 4
```

-> 从 `%4` 里读取 `i`。  
-> 所以：

```text
%10 = i
```

---

```llvm
  %11 = load i32, ptr %3, align 4
```

-> 从 `%3` 里读取 `s`。  
-> 所以：

```text
%11 = s
```

---

```llvm
  %12 = add nsw i32 %11, %10
```

-> 计算 `%11 + %10`。  
-> `%11` 是 `s`。  
-> `%10` 是 `i`。  
-> 所以：

```text
%12 = s + i
```

-> `add` 是加法。  
-> `nsw` 表示 no signed wrap，也就是 LLVM 假设这个有符号加法不会溢出。  
-> 对应：

```c
s + i
```

---

```llvm
  store i32 %12, ptr %3, align 4
```

-> 把 `%12` 存回 `%3` 指向的内存。  
-> `%3` 是变量 `s` 的地址。  
-> 所以：

```c
s = s + i;
```

也就是：

```c
s += i;
```

---

```llvm
  br label %13
```

-> 无条件跳转到基本块 `%13`。  
-> `%13` 是循环更新块，也就是 `i++` 的位置。

---

```llvm
13:                                               ; preds = %9
```

-> 这是基本块 `%13`。  
-> 它从 `%9` 循环体跳过来。  
-> 对应 C 里的：

```c
i++;
```

---

```llvm
  %14 = load i32, ptr %4, align 4
```

-> 从 `%4` 里读取当前的 `i`。  
-> 所以：

```text
%14 = i
```

---

```llvm
  %15 = add nsw i32 %14, 1
```

-> 计算 `%14 + 1`。  
-> `%14` 是当前的 `i`。  
-> 所以：

```text
%15 = i + 1
```

-> 对应：

```c
i + 1
```

---

```llvm
  store i32 %15, ptr %4, align 4
```

-> 把 `%15` 存回 `%4`。  
-> `%4` 是变量 `i` 的地址。  
-> 对应：

```c
i = i + 1;
```

也就是：

```c
i++;
```

---

```llvm
  br label %5, !llvm.loop !6
```

-> 无条件跳回 `%5`。  
-> `%5` 是循环条件判断块。  
-> 所以执行完 `i++` 后，回去重新判断：

```c
i <= n
```

-> `!llvm.loop !6` 是循环元数据，初学可以先忽略。它只是告诉 LLVM：“这里是一个循环”。

---

```llvm
16:                                               ; preds = %5
```

-> 这是基本块 `%16`。  
-> 它是循环退出块。  
-> 当 `%5` 里判断 `i <= n` 为 false 时，就会跳到这里。  
-> 对应：

```c
循环结束
```

---

```llvm
  %17 = load i32, ptr %3, align 4
```

-> 从 `%3` 里读取 `s` 的最终值。  
-> 所以：

```text
%17 = s
```

---

```llvm
  ret i32 %17
```

-> 返回 `%17`。  
-> `%17` 是最终的 `s`。  
-> 对应：

```c
return s;
```

---

```llvm
}
```

-> 函数结束。

### 未优化 IR 翻译成 C 风格伪代码

```c
int sum_to(int n) {
    int s = 0;
    int i = 1;

loop_condition:
    if (i <= n)
        goto loop_body;
    else
        goto loop_exit;

loop_body:
    s = s + i;
    goto loop_increment;

loop_increment:
    i = i + 1;
    goto loop_condition;

loop_exit:
    return s;
}
```

最关键的对应关系是：

```text
%2 = n 的地址
%3 = s 的地址
%4 = i 的地址

%5  = 判断 i <= n
%9  = 执行 s += i
%13 = 执行 i++
%16 = return s
```

## 2. 优化后的 IR 逐行讲解

优化后已经没有 `s`、`i`、循环了。LLVM 把：

```c
for (int i = 1; i <= n; i++) {
    s += i;
}
```

优化成了一个数学公式：

```text
1 + 2 + ... + n
```

也就是：

```c
n * (n + 1) / 2
```

但 LLVM 生成的公式看起来稍微绕一点，是为了处理溢出和整数位宽。

```llvm
define dso_local i32 @sum_to(i32 noundef %0) local_unnamed_addr #0 {
```

-> 定义函数 `sum_to`。  
-> 返回值是 `i32`。  
-> 参数 `%0` 就是原来的 `n`。  
-> `local_unnamed_addr #0` 是函数属性相关，先不用管。

对应：

```c
int sum_to(int n) {
```

---

```llvm
  %.not7 = icmp slt i32 %0, 1
```

-> 比较 `%0 < 1`。  
-> `%0` 是 `n`。  
-> `icmp` 是整数比较。  
-> `slt` 是 signed less than，有符号小于。  
-> 所以：

```text
%.not7 = n < 1
```

对应 C：

```c
if (n < 1)
```

为什么先判断 `n < 1`？因为原始循环是：

```c
for (int i = 1; i <= n; i++)
```

如果 `n < 1`，循环一次都不会执行，直接返回 `0`。

补充：`%.not7` 是 LLVM IR 里的一个临时值名字。它真正的局部名字标志是前面的 `%`，`.` 只是名字的一部分。可以把它脑内改名成：

```llvm
%is_n_less_than_1 = icmp slt i32 %0, 1
```

---

```llvm
  br i1 %.not7, label %._crit_edge, label %.lr.ph.preheader
```

-> 根据 `%.not7` 跳转。  
-> 如果 `n < 1` 为 true，跳到 `%._crit_edge`。  
-> 如果 `n < 1` 为 false，也就是 `n >= 1`，跳到 `%.lr.ph.preheader`。

对应：

```c
if (n < 1)
    goto exit;
else
    goto compute_formula;
```

---

```llvm
.lr.ph.preheader:                                 ; preds = %1
```

-> 这是一个基本块，名字是 `%.lr.ph.preheader`。  
-> 它负责计算优化后的数学公式。  
-> `preds = %1` 表示它从入口块跳过来。  
-> `%1` 是入口基本块的隐式编号，也就是刚才做 `n < 1` 判断的地方。

名字不用深究，`lr.ph.preheader` 可以粗略理解成：

```text
原来循环开始前的准备块
```

但现在循环已经被优化没了，这里只剩公式计算。

---

```llvm
  %2 = shl nuw i32 %0, 1
```

-> 把 `%0` 左移 1 位。  
-> `%0` 是 `n`。  
-> `shl` 是 shift left，左移。  
-> 左移 1 位等价于乘以 2。  
-> 所以：

```text
%2 = n * 2
```

也就是：

```c
int tmp2 = n << 1;  // 2n
```

`nuw` 是 no unsigned wrap，表示无符号意义下不会溢出。这里先记住它是优化器的安全标记。

---

```llvm
  %3 = add nsw i32 %0, -1
```

-> 计算 `%0 + (-1)`。  
-> 也就是：

```text
%3 = n - 1
```

对应：

```c
int tmp3 = n - 1;
```

`nsw` 表示 no signed wrap，有符号加法不会溢出。

---

```llvm
  %4 = zext nneg i32 %3 to i33
```

-> 把 `%3` 从 `i32` 扩展成 `i33`。  
-> `%3` 是 `n - 1`。  
-> `zext` 是 zero extend，零扩展。  
-> 所以：

```text
%4 = 把 n - 1 扩展成 33 位整数
```

为什么是 `i33`？因为后面要做乘法，LLVM 用 33 位来减少中间计算溢出的风险。

`nneg` 表示这个值非负。因为前面已经保证进入这个块时 `n >= 1`，所以：

```text
n - 1 >= 0
```

---

```llvm
  %5 = add nsw i32 %0, -2
```

-> 计算：

```text
%5 = n - 2
```

对应：

```c
int tmp5 = n - 2;
```

---

```llvm
  %6 = zext i32 %5 to i33
```

-> 把 `%5` 从 `i32` 扩展成 `i33`。  
-> `%5` 是 `n - 2`。  
-> 所以：

```text
%6 = 把 n - 2 扩展成 33 位整数
```

这里注意：如果 `n = 1`，那么 `n - 2 = -1`。  
`zext` 会把它当成无符号位模式扩展，所以这块看起来怪。LLVM 这么写是基于更底层的整数环绕语义和后续截断，初学先不要被它吓住。

---

```llvm
  %7 = mul i33 %4, %6
```

-> 计算 `%4 * %6`。  
-> `%4` 是扩展后的 `n - 1`。  
-> `%6` 是扩展后的 `n - 2`。  
-> 所以大概是：

```text
%7 = (n - 1) * (n - 2)
```

注意这是 `i33` 乘法，不是 `i32`。

---

```llvm
  %8 = lshr i33 %7, 1
```

-> 对 `%7` 做逻辑右移 1 位。  
-> `lshr` 是 logical shift right。  
-> 对非负整数来说，右移 1 位等价于除以 2。  
-> 所以可以理解成：

```text
%8 = ((n - 1) * (n - 2)) / 2
```

---

```llvm
  %9 = trunc nuw i33 %8 to i32
```

-> 把 `%8` 从 `i33` 截断回 `i32`。  
-> `trunc` 是 truncate，截断。  
-> 所以：

```text
%9 = 把中间结果转回 32 位整数
```

`nuw` 还是和无符号不溢出相关的标记。

---

```llvm
  %10 = add i32 %2, %9
```

-> 计算：

```text
%10 = %2 + %9
```

前面：

```text
%2 = 2n
%9 = ((n - 1) * (n - 2)) / 2
```

所以：

```text
%10 = 2n + ((n - 1) * (n - 2)) / 2
```

---

```llvm
  %11 = add i32 %10, -1
```

-> 计算：

```text
%11 = %10 - 1
```

代入上面就是：

```text
%11 = 2n + ((n - 1) * (n - 2)) / 2 - 1
```

这个式子看起来和 `n * (n + 1) / 2` 不一样，但其实等价：

```text
2n - 1 + ((n - 1)(n - 2))/2
= (4n - 2 + n^2 - 3n + 2) / 2
= (n^2 + n) / 2
= n(n + 1) / 2
```

所以 `%11` 就是：

```text
1 + 2 + ... + n
```

---

```llvm
  br label %._crit_edge
```

-> 公式算完后，跳到 `%._crit_edge`。  
-> `%._crit_edge` 是统一返回的位置。

---

```llvm
._crit_edge:                                      ; preds = %.lr.ph.preheader, %1
```

-> 这是返回前的汇合块。  
-> 它可能从两个地方来：

```text
%1                 入口块，也就是 n < 1 的情况
%.lr.ph.preheader  公式计算块，也就是 n >= 1 的情况
```

---

```llvm
  %.06.lcssa = phi i32 [ 0, %1 ], [ %11, %.lr.ph.preheader ]
```

-> 这是一个 `phi` 指令。  
-> `phi` 的作用是：根据“从哪个基本块跳过来”，选择不同的值。

这句意思是：

```text
如果从 %1 过来，结果是 0
如果从 %.lr.ph.preheader 过来，结果是 %11
```

也就是：

```c
if (n < 1)
    result = 0;
else
    result = formula_result;
```

其中 `%11` 是刚才算出来的：

```text
n(n + 1) / 2
```

所以整体就是：

```c
result = (n < 1) ? 0 : n * (n + 1) / 2;
```

---

```llvm
  ret i32 %.06.lcssa
```

-> 返回 `%.06.lcssa`。  
-> 如果 `n < 1`，返回 `0`。  
-> 如果 `n >= 1`，返回 `%11`，也就是 `1 + 2 + ... + n`。

---

```llvm
}
```

-> 函数结束。

### 优化后的 IR 翻译成 C 风格伪代码

```c
int sum_to(int n) {
    if (n < 1) {
        return 0;
    } else {
        return 2 * n + ((n - 1) * (n - 2)) / 2 - 1;
    }
}
```

这个式子等价于：

```c
int sum_to(int n) {
    if (n < 1) {
        return 0;
    } else {
        return n * (n + 1) / 2;
    }
}
```

所以你可以这样总结优化后的 IR：

```text
原来：
    s = 0;
    for (i = 1; i <= n; i++)
        s += i;
    return s;

优化后：
    if (n < 1)
        return 0;
    return n(n + 1) / 2;
```

最关键的几个值：

```text
%0          = n
%.not7      = n < 1
%2          = 2n
%3          = n - 1
%5          = n - 2
%7          = (n - 1) * (n - 2)
%8          = ((n - 1) * (n - 2)) / 2
%11         = n(n + 1) / 2
%.06.lcssa  = 最终返回值，n < 1 时是 0，否则是 %11
```
